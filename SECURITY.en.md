# Security Policy

## Reporting a vulnerability

**Do not open public issues for security vulnerabilities.** Use one of the two private
channels below:

1. **GitHub Security Advisory**: repository → **Security** → **Report a vulnerability**.
   Opens a private advisory visible only to the maintainers and the MH SecOps team.
2. **Malakoff Humanis Vulnerability Disclosure Program (VDP)**: per the
   [MH vulnerability disclosure policy](https://www.malakoffhumanis.com/politique-divulgation-vulnerabilites/),
   submit your report via the [VDP form](https://vdp.malakoffhumanis.com/p/Send-a-report).
   **The report title must start with `[baldr opensource]`.**

## Process

1. **Acknowledgement within 72 hours** of the report.
2. **Triage and qualification by the MH SecOps team**, under embargo.
3. Fix developed by the maintainers in a private fork.
4. A **GitHub Security Advisory** is published (with a CVE where applicable).
5. **Coordinated disclosure** once a fix is released.
