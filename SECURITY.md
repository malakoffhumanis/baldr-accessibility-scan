# Politique de sécurité

## Signaler une vulnérabilité

**Ne pas ouvrir d'issues publiques pour les vulnérabilités de sécurité.** Utilisez l'un
des deux canaux privés ci-dessous :

1. **GitHub Security Advisory** : dépôt → **Security** → **Report a vulnerability**.
   Ouvre un advisory privé visible uniquement par les mainteneurs et l'équipe SecOps MH.
2. **Programme de divulgation de vulnérabilités (VDP) de Malakoff Humanis** :
   conformément à la [politique de divulgation des vulnérabilités de MH](https://www.malakoffhumanis.com/politique-divulgation-vulnerabilites/),
   soumettez votre rapport via le [formulaire VDP](https://vdp.malakoffhumanis.com/p/Send-a-report).
   **Le titre du rapport doit commencer par `[baldr opensource]`.**

## Processus

1. **Accusé de réception sous 72 h** à compter du rapport.
2. **Triage et qualification par l'équipe SecOps MH**, sous embargo.
3. Correctif développé par les mainteneurs dans un fork privé.
4. Une **GitHub Security Advisory** est publiée (avec un CVE le cas échéant).
5. **Divulgation coordonnée** une fois le correctif publié.
